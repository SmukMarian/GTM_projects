# Projects Tracker — Project Plan

## 1. Цели релиза v1

Цель релиза v1 — полнофункциональная рабочая версия локального веб-приложения Projects Tracker, реализующая все требования из `Haier_Project_Tracker_TZ.md`:

- полноценный дашборд;
- продуктовые группы и проекты;
- GTM-этапы;
- задачи и подзадачи;
- характеристики (RU/EN);
- галерея и обложка;
- файлы и комментарии;
- история изменений;
- архивирование и удаление;
- контекстные меню;
- импорт/экспорт через Excel;
- резервное копирование.

MVP не нужен — сразу делаем боевую версию.

---

## 2. Этапы проекта

### Этап 1. Анализ ТЗ и проектирование

**Задачи:**

- Ещё раз внимательно пройти `Haier_Project_Tracker_TZ.md`.
- Составить текстовую модель данных (перечень сущностей и связей).
- Описать основные экраны:
  - Дашборд;
  - экран продуктовой группы;
  - карточка проекта (со всеми вкладками);
  - представления задач и подзадач;
  - экраны импорта/экспорта;
  - экран резервного копирования.

**Результат:**

- Зафиксированная схема сущностей (текст или диаграмма).
- Понимание всех экранов и сценариев навигации.

**Критерий готовности:**

- Сущности и связи совпадают с ТЗ.
- Нет «висящих» требований из ТЗ, для которых не понятно, где они будут реализованы.

---

### Этап 2. Базовая инфраструктура приложения

**Задачи:**

- Создать структуру проекта (директории, базовые модули).
- Реализовать базовый слой хранения данных (единый источник данных).
- Добавить механизм конфигурации для:
  - пути хранения основного файла данных;
  - пути для резервных копий.

**Результат:**

- Проект собирается и запускается локально.
- Приложение открывается по локальному адресу в браузере.
- Есть базовые заглушки экранов (страницы открываются, даже если пока без всех деталей).

**Критерий готовности:**

- У пользователя есть рабочий локальный запуск.
- Основная структура проекта стабильна.

---

### Этап 3. Реализация сущностей и операций

Этот этап делится на подэтапы по сущностям.

#### 3.1. Продуктовые группы

**Задачи:**

- Реализовать модель «Продуктовая группа».
- Реализовать создание, редактирование, архивирование, восстановление.
- Реализовать удаление (с подтверждением и понятной логикой по связанным проектам).
- Реализовать экран списка групп:
  - отображение сводной информации;
  - фильтры по статусу, бренду и др.;
  - контекстное меню по правой кнопке.

**Критерий готовности:**

- Можно создать несколько групп, отфильтровать, архивировать/вернуть и удалить.

#### 3.2. Проекты

**Задачи:**

- Реализовать модель «Проект» с полями из ТЗ.
- Реализовать:
  - создание проекта в выбранной группе;
  - редактирование;
  - смену статусов (Активный / Закрыт / Архивирован);
  - архивирование и восстановление;
  - удаление (со всеми дочерними сущностями).
- Реализовать список проектов в группе:
  - карточки и табличное представление;
  - фильтры по бренду, статусу, GTM-этапу, периоду запуска;
  - контекстное меню по правой кнопке.

**Критерий готовности:**

- Можно вести несколько проектов в разных группах и управлять их жизненным циклом.

#### 3.3. Go-to-market этапы

**Задачи:**

- Реализовать модель GTM-этапа.
- Реализовать экран управления этапами в карточке проекта:
  - добавление, редактирование, удаление;
  - изменение порядка;
  - статусы, даты, признак риска;
  - чек-лист внутри этапа.
- Реализовать работу с шаблонами GTM-этапов:
  - сохранить структуру этапов как шаблон;
  - применить шаблон при создании проекта;
  - опционально — применить к существующему проекту.

**Критерий готовности:**

- Для проекта можно полностью настроить цепочку этапов и отмечать прогресс.

#### 3.4. Задачи и подзадачи

**Задачи:**

- Реализовать модели «Задача» и «Подзадача».
- Реализовать:
  - создание, редактирование, смену статуса;
  - привязку к GTM-этапу;
  - подзадачи как чек-лист;
  - комментарии к задачам.
- Реализовать представления:
  - список задач;
  - доска по статусам.
- Реализовать фильтры:
  - по статусу, срокам, GTM-этапу.
- Реализовать контекстные меню для задач и подзадач.

**Критерий готовности:**

- Для проекта можно завести набор задач с подзадачами, отмечать прогресс и видеть их в разных представлениях.

#### 3.5. Характеристики продукта (RU/EN)

**Задачи:**

- Реализовать структуру секций и полей характеристик.
- Обеспечить отображение Label/Value RU и EN рядом.
- Реализовать:
  - добавление/удаление/переименование секций;
  - добавление/удаление/редактирование полей;
  - настройку типов полей;
  - изменение порядка.
- Реализовать шаблоны характеристик:
  - создание шаблона;
  - применение при создании нового проекта;
  - копирование структуры из существующего проекта.

**Критерий готовности:**

- Можно создать структуру характеристик для продукта, заполнить RU/EN значения и переиспользовать структуру.

#### 3.6. Галерея и обложка

**Задачи:**

- Реализовать хранение и отображение галереи изображений в проекте.
- Реализовать:
  - загрузку изображений;
  - подписи/комментарии;
  - изменение порядка;
  - выбор обложки;
  - удаление.
- Обеспечить использование обложки в списках и карточках проектов.

**Критерий готовности:**

- Для проекта можно загрузить несколько изображений и выбрать одно как обложку.

#### 3.7. Файлы и комментарии

**Задачи:**

- Реализовать работу с файлами:
  - загрузка, описание, категория, дата;
  - скачивание;
  - удаление.
- Реализовать комментарии к проекту:
  - добавление, отображение;
  - при необходимости — редактирование и удаление.

**Критерий готовности:**

- Можно прикреплять и управлять файлами и комментариями в проекте.

#### 3.8. История изменений

**Задачи:**

- Определить, какие события точно логируем (по ТЗ).
- Реализовать механизм записи истории.
- Реализовать отображение истории изменений в карточке проекта.

**Критерий готовности:**

- Основные операции по проекту отражаются в ленте истории.

---

### Этап 4. Импорт/экспорт

**Задачи:**

- Реализовать экспорт и импорт характеристик проекта (RU/EN) через Excel.
- Реализовать экспорт и импорт GTM-структуры и шаблонов.
- Реализовать экспорт списка проектов с ключевыми полями.
- Обработать ошибки сопоставления и сделать сообщения понятными.

**Критерий готовности:**

- Все заявленные сценарии импорта/экспорта работают и дают ожидаемый результат.

---

### Этап 5. Архивирование, удаление и контекстные меню

**Задачи:**

- Проверить, что для всех сущностей из ТЗ реализованы:
  - статусы (активный/закрытый/архивный — где нужно);
  - архивирование/восстановление;
  - удаление с подтверждением.
- Проверить, что во всех нужных местах реализованы контекстные меню по правой кнопке мыши с ключевыми действиями.

**Критерий готовности:**

- Любую сущность, у которой в ТЗ указано архивирование и/или удаление, реально можно заархивировать/удалить через UI и контекстные меню.

---

### Этап 6. Резервное копирование

**Задачи:**

- Реализовать:
  - создание резервной копии базы (экспорт в файл);
  - восстановление из резервной копии.
- Проверить сценарий ручного копирования основного файла данных.

**Критерий готовности:**

- Можно сохранить состояние системы в отдельный файл и восстановить его.

---

### Этап 7. Финальная шлифовка и тестирование

**Задачи:**

- Пройтись по `Haier_Project_Tracker_TZ.md` и сверить, что все пункты реализованы.
- Проверить основные пользовательские сценарии:
  - создание/ведение групп и проектов;
  - настройка GTM-этапов;
  - работа с задачами/подзадачами;
  - заполнение характеристик RU/EN;
  - работа с галереей, файлами, комментариями;
  - импорт/экспорт;
  - архивирование/удаление;
  - резервные копии.
- Исправить найденные баги.

**Критерий готовности:**

- Все требования ТЗ либо выполнены, либо явно помечены как обсуждаемые изменения с пользователем.
