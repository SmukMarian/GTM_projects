# Haier Project Tracker — Дополнительные правки UI/UX (волна 3)

Отмечай выполненные пункты, заменяя `[ ]` на `[x]`.

---

## 1. Дашборд

- [x] Увеличить вертикальные отступы между основными блоками
  - Для карточек/секций использовать отступы порядка `margin: 10px 0;` или больше, чтобы блоки не слипались.
  - Проверить, что внутри каждой секции тоже есть комфортные внутренние отступы (`padding`).

- [x] Обновить внешний вид чекбоксов на дашборде
  - Сделать сам квадрат чекбокса немного крупнее и визуально аккуратнее.
  - Привести чекбоксы к единому современному стилю (ровные границы, внятная галочка, нормальные отступы от подписи).

- [x] Переработать блок «Последние изменения»
  - Отображать изменения в **2 колонки**.
  - На одной «странице» показывать максимум **8 изменений** (4 строки × 2 столбца).
  - Ниже добавить кнопку «Показать ещё», которая подгружает следующую «страницу» из 8 записей.
  - Общий лимит: не показывать более **24 изменений**; если их больше, старые записи (старше 1 месяца) не подгружать.
    - Логика: backend отдаёт последние изменения за последние 30 дней; фронт разбивает по 8 на шаг, максимум 3 шага (24 записи).

- [x] Удалить лишние ссылки-бейджи, заменённые меню
  - Убрать с дашборда:
    - `<a href="/groups.html" class="badge" ...>Продуктовые группы →</a>`
    - `<a href="/projects.html" class="badge" ...>Проекты →</a>`
  - Перемещение к группам и проектам выполняется через верхнее меню.

- [x] Обновить внешний вид выпадающих списков

    - плавные скругления;

- [x] Обновить чекбоксы и кнопки «Применить / Сбросить»
  - Применить тот же стиль чекбоксов, что и на дашборде.
  - Кнопки «Применить фильтры» и «Сбросить» оформить так же, как на дашборде:
    - основная зелёная кнопка для применения;

- [x] Изменить раскладку групп — 2 карточки в ряд
  - Отображать продуктовые группы сеткой **по 2 карточки в ряд**.

- [x] Добавить быстрый просмотр группы (popup)

- [x] Унифицировать работу с данными только через pop-up
  - Создание/редактирование группы, изменение параметров должны выполняться в единообразных модальных окнах (pop-up), а не в россыпь по странице.

- [x] Удалить лишние текстовые ссылки навигации
  - Убрать:
    - `<a href="/" title="К дашборду">⟵ Дашборд</a>`

## 4. Карточка проекта

- [x] Исправить выравнивание обложки проекта
  - Сейчас фон/контейнер обложки вылезает за разметку.
  - Требуется:
    - задать фиксированный контейнер для обложки;
    - настроить `object-fit: cover` для изображения;
    - убедиться, что изменение `min-height` в `.project-hero-right` больше не ломает вёрстку (либо убрать зависимость от `min-height: 100%` корректным способом).

- [x] Вынести редактирование основных параметров в pop-up
  - Основные параметры (бренд, группа, рынок, даты, статус, цены, приоритет и т.п.) сейчас смещены и выглядят отдельным блоком.
  - Решение:
    - в шапке проекта слева оставить только отображение ключевых полей;
    - рядом с названием модели добавить иконку карандаша;
    - по клику открывать модальное окно шириной ~800px с формой редактирования всех основных параметров.

- [ ] Объединить GTM-этапы, задачи и подзадачи в единый блок
  - Единая логика:
    - GTM-этап — верхний уровень;
    - внутри этапа — задачи;
    - внутри задач — подзадачи.
  - Требуется:
    - визуально объединить всё в один блок GTM;
    - сделать общий статус-бар для всего GTM по проекту и отдельные прогресс-бары по каждому этапу;
    - прогресс этапа считать от задач, прогресс задачи — от подзадач;
    - общая логика выполнения: всё считается в одной связке, а не как разрозненные сущности.

- [ ] Упростить структуру GTM-этапов
  - Внутренние «пункты» внутри GTM-этапов больше не нужны как отдельная сущность.
  - Для каждого этапа оставить:
    - заголовок;
    - описание (с текстовым редактором);
    - комментарии к этапу.

- [x] Убрать блок «Характеристики» из карточки проекта
  - Поскольку характеристики вынесены на отдельную страницу, блок внутри основной карточки проекта больше не показывать.

- [ ] Поправить попап создания/редактирования задач
  - Ширина ~**800px**, по центру экрана.
  - Поля разнести с нормальными отступами (название, описание, GTM-этап, даты, чекбоксы и т.д.).
  - Чекбоксы привести к единому аккуратному стилю, как на фильтрах.

- [ ] Улучшить блок комментариев в задаче
  - Отделить комментарии от формы добавления (отступы сверху/снизу).
  - Кнопку «Добавить» выровнять относительно поля ввода и других элементов.

- [ ] Загрузки файлов — через pop-up
  - При добавлении файлов к задаче/проекту:
    - показывать модальное окно с выбором файла, названием, описанием;
    - не располагать форму «вросшей» в основную страницу.

- [ ] Загрузка фото и галереи — не ухудшать качество
  - При загрузке изображений не сжимать их до низкого качества.
  - Если нужна оптимизация, использовать компрессию без заметной потери качества и хранить исходник.
